# Spec Interpretation: Task 2 -- テスタビリティレビュー追加

## 対象要件
- Requirement: REQ-001 テスタビリティレビューの追加
  - **仕様の記述**:
    - Happy Path 1: GIVEN brainstorm.md にステップ5.8 が定義されている WHEN ステップ5.8 の内容を確認する THEN 各ユーザーストーリーに「このストーリーはどう検証しますか？」と問いかけるワークフローが記述されている
    - Happy Path 2: GIVEN ステップ5.8 でユーザーが検証観点を回答した WHEN proposal.md のユーザーストーリーを確認する THEN 各ストーリーの下に「検証観点: [回答]」が追記されている
    - Happy Path 3: GIVEN ステップ5.8 でユーザーが曖昧な検証観点を回答した WHEN ステップの記述を確認する THEN より具体的な検証方法を追加質問するよう指示されている
    - Error 1: GIVEN ステップ5.8 でユーザーが「このまま進める」と判断した WHEN 検証観点が不十分なストーリーがある THEN ステップは非ブロッキングであり、そのまま次のステップに進む
    - Error 2: GIVEN ステップ5.8 が実行される WHEN proposal.md にユーザーストーリーが0件である THEN テスタビリティレビューをスキップする
    - Boundary 1: GIVEN ステップ5.8 が実行される WHEN ユーザーストーリーが1件のみ存在する THEN その1件に対して検証観点を問いかける
    - NFR: 既存のステップ5.7 の動作・出力を変更しない
  - **私の解釈**: brainstorm.md の行47（ステップ5.7 の最終行）の直後にステップ5.8 のテキストブロックを挿入する。提案書テンプレート（行88）のユーザーストーリー行の下に「検証観点: [どう検証するか]」行を追加する。タスク説明に記載された具体テキストをそのまま使用する
  - **前提（仕様に未記載だが私が仮定すること）**: ステップ5.8 の追加により後続のステップ番号（6, 7, 7.5）は繰り下げ不要（既に6以降であり5.x系の連番に影響しない）
  - **仕様でカバーされないエッジケース**: 検証観点の文言フォーマット（長さ制限等）は未定義 -- brainstorm は自由な対話のため制約は不要と判断
